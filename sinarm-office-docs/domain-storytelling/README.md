# Domain Storytelling

## Objetivo

Desenhar como o escritorio funciona no mundo real antes de pensar em banco,
PostgreSQL, Django ou telas.

Esta etapa revela:

- Eventos.
- Comandos.
- Entidades.
- Agregados.
- Filas.
- Notificacoes.
- Prazos.
- Pontos de auditoria.

## Escopo Inicial

- Aquisicao.
- Renovacao.
- Transferencia.
- Porte.

## Regra

Nenhum `models.py`, migration ou estrutura de banco deve ser criada a partir
desta etapa sem aprovacao dos fluxos de negocio.
