# Release Policy

## Politica

- Release deve passar por testes automatizados.
- Migrations devem ser revisadas.
- Mudancas de dominio devem estar documentadas.
- Mudancas de seguranca devem ser avaliadas.
- Backup e rollback devem ser considerados.

## Regra

Nenhuma release deve introduzir alteracao estrutural sem documentacao e revisao.
